# Unlimited Referral System - 20% Profit Allocation

## 1. System Goal

Design a **global referral economy** inside the delivery platform where:

- **20% of platform profit** is allocated to a **referral reward pool**
- The referral tree has **unlimited depth**
- Every ancestor in the chain receives a **decreasing share**
- The **total payout never exceeds 20%**

This creates viral platform growth, passive income incentives, and decentralized marketing.

## 2. Core Economic Rule

Platform profit from an order:
```
platform_profit = order_fee - operational_costs
```

Referral pool:
```
referral_pool = platform_profit * 0.20
```

Example:
```
Order value: $100
Platform commission: $20
Referral pool: $4
```
The **$4 is distributed across the referral chain**.

## 3. Distribution Model

Use **geometric decay**:
```
reward(level) = referral_pool * (1/2^level)
```
Example distribution:

| Level | Share of referral pool |
|-------|------------------------|
| 1 | 50% |
| 2 | 25% |
| 3 | 12.5% |
| 4 | 6.25% |
| 5 | 3.125% |
| ... | continues |

Total sum ≈ 100% of referral_pool (i.e., 20% of platform profit).

## 4. Example Real Order

## 8. Order Completion Flow
1. Calculate platform profit
2. Create referral pool
3. Traverse referral chain
4. Allocate rewards
5. Update wallets

Flow:
```
Order Completed → Calculate Platform Profit → Referral Pool → Traverse Referral Chain → Distribute Rewards → Update Wallet Balances
```

## 9. Reward Algorithm (Pseudo code)
```
pool = platform_profit * 0.20
level = 1
share = 0.5
current_user = order_user.referrer

while current_user exists:
    reward = pool * share
    if reward < MIN_REWARD:
        break
    credit(current_user, reward)
    share = share / 2
    level += 1
    current_user = current_user.referrer
```

## 10. Performance Optimization
- **Minimum reward cutoff:** reward < $0.0001 → stop
- **Background worker:** enqueue payout job using BullMQ/RabbitMQ/Redis queue

## 11. Fraud Protection
