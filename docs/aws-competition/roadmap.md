## Phase 2: AWS Model Training (High Priority)

_Duration: 1-2 Days (Mostly waiting for training)_

### 1. Amazon Forecast (Demand Prediction)

- **Action:** Create Dataset Group `MatrixOrderDemand`.
- **Schema:** `timestamp`, `item_id` (zone), `target_value` (order_count).
- **Train:** Train a predictor using `AutoML` (it will pick the best algorithm like DeepAR+).
- **Output:** Enable "Create Forecast" to generate predictions for the next 48 hours.

### 3. Amazon Personalize (Intelligent Matching)

- **Action:** Create Dataset Group `MatrixDriverMatching`.
- **Schema:** `USER_ID`, `ITEM_ID`, `TIMESTAMP`, `EVENT_TYPE`.
- **Train:** Train a Solution using `User-Personalization` recipe.
- **Campaign:** Deploy a campaign to get real-time recommendations (e.g., "Top 5 drivers for Zone X").

### 4. Amazon Fraud Detector (Security)

- **Action:** Define an entity `order` and event `transaction`.
- **Train:** Upload a dataset of "good" vs "bad" transactions (we may need to generate a small fraud CSV for this).
- **Rule:** Create a rule: `IF risk_score > 90 THEN decline_order`.

## Phase 3: The "Brain" - Serverless Integration

_Duration: 2-3 Days_

We will not call AWS directly from the mobile app (security risk). We will use AWS Lambda.

1.  **Lambda Function `getPredictedDemand`:**
    - Queries Amazon Forecast.
    - Returns JSON: `{ "zone_downtown": "HIGH", "zone_suburbs": "LOW" }`.
2.  **Lambda Function `getBestDrivers`:**
    - Input: `order_zone`.
    - Queries Amazon Personalize.
    - Returns: `[driver_id_1, driver_id_5, ...]` (sorted by compatibility).

3.  **API Gateway:**
    - Expose these Lambdas as REST endpoints:
    - `GET /ai/demand`
    - `POST /ai/match`

## Phase 4: Backend Integration (The "Nervous System")

_Duration: 2 Days_

Modify our Node.js Backend (`backend/services/orderService.js`):

1.  **On Order Creation:**

2.  **On App Launch (Driver):**
