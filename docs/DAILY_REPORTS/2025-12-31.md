## 🔐 1. Security Hardening (Sprints 2-5)

### Token Revocation System (Sprint 5)
Implemented **JWT blacklisting** using Redis:
- On logout, token signature stored in Redis with TTL matching token's remaining lifetime
- Every authenticated request checks Redis for revoked tokens
- **Files**: `authService.js`, `middleware/auth.js`, `config/redis.js`

### Device-Aware Rate Limiting
Implemented **FingerprintJS-based rate limiting**:
- Frontend generates unique device fingerprint via `x-device-fingerprint` header
- Rotating IPs doesn't bypass limits—the *device* is tracked

### IDOR Fix
Fixed **Insecure Direct Object Reference** on `/orders/:id`:
- Added strict ownership verification

### Critical Server Startup Bug
Fixed Redis client returning `null` during startup:
- **Root Cause**: `server.js` imported modules before loading `.env`
- **Fix**: Moved `dotenv.config()` to top of `server.js`
